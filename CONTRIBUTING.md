# Contributing

This repository models a lightweight team workflow.

## Workflow
1. Create a branch for each change.
2. Open a PR into `main`.
3. Ensure required checks pass (when configured).
4. Squash-merge the PR.

## Branch naming
- docs/<topic>
- chore/<task>
- feat/<feature>
- fix/<bug>
- ci/<pipeline-change>

## Commit messages
Keep messages descriptive and scoped.
